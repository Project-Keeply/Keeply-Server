# Debrief Format

작업 완료 후(사후 브리핑) 아래 형식으로 보고합니다.

## 표준 템플릿

     작업 완료

    ### 컨텍스트
    - 브랜치: {브랜치명}
    - 이슈: #{번호} — {제목}
    - 스킬: {사용한 스킬명}

    ### 변경 사항
    - 생성: {파일 경로}
    - 수정: {파일 경로}
    - 삭제: {파일 경로}
    - 총 {N}개 파일, +{X}/-{Y} lines

    ### 검증 결과
    - Spotless: PASS / FAIL / N/A
    - Compile: PASS / FAIL / N/A
    - Test: PASS / FAIL / N/A
    - Build: PASS / FAIL / N/A
    - Swagger 확인: PASS / FAIL / N/A

    ### 커밋
    - {커밋 해시} {커밋 메시지}

    ### Docs 영향
    - 업데이트 필요: {있음/없음}
    - 대상: {문서 경로}

    ### 다음 스텝 제안
    - {제안 1}
    - {제안 2}

## 원칙

- 파일 경로는 항상 명시
- 검증 못 한 항목은 "N/A" 또는 "확인 못 함"으로 표기
- 성공/실패를 명확히 (모호한 "완료됨" 금지)
- 다음 스텝은 항상 제시
